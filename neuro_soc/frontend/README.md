# Neuro-SOC — Frontend (SOC Analyst Dashboard)

> The **Human-in-the-Loop** command centre for Neuro-SOC. A dark-themed, enterprise-grade Security Operations Center dashboard built with **Next.js 16**, **React 19**, and **TailwindCSS 4**.

---

## Overview

The frontend is Tier 3 of the Neuro-SOC detection pipeline — the **final authority** layer. It surfaces ML-scored anomalies with full SHAP explainability and LLM-generated threat narratives, empowering SOC analysts to make informed quarantine or dismiss decisions in seconds.

> **Key Design Principle:** The system *cannot* quarantine a user without analyst confirmation. The ML engine scores, the LLM translates, and only a human presses the red button. This satisfies GDPR Article 22's prohibition on fully automated enforcement.

## Features

### Sidebar — Event List Panel

- **Real-Time Model Metrics** — Live Precision, Recall, and F1 scores computed against ground-truth labels
- **Severity-Coded Event Cards** — Each flagged event is color-coded by severity (Critical / High / Medium) with risk score badges
- **Filter Tabs** — Quick-filter events by severity level (All, Critical, High, Medium) with live counts
- **Refresh Control** — Force recomputation of the ML pipeline from the dashboard

### Main Panel — Investigation View

- **Risk Score Gauge** — 0–100 score with severity label, color-coded progress bar, and raw anomaly score
- **Recommended Action Badge** — LLM-derived action (Quarantine / Investigate / Monitor) with threshold-based color coding
- **Ground Truth Comparison** — Side-by-side view of ML prediction vs. labelled ground truth (anomaly type, severity, explanation)
- **Threat Summary** — Natural-language narrative generated by the constrained Llama-3 LLM, explaining *why* the event was flagged
- **Evidence List** — Three plain-English evidence bullets derived from SHAP feature attributions
- **SHAP Feature Weights** — Visual bar chart of top contributing features with normalized bars and raw SHAP values
- **Event Context** — Data asset, sensitivity level, department, query type, destination, and timestamp
- **Analyst Decision** — Two action buttons:
  - 🔴 **Quarantine User & Revoke Access** — Immediate enforcement
  - ✅ **Mark as False Positive** — Dismiss with full audit log

## API Integration

The frontend communicates with the FastAPI backend on `http://localhost:8000`:

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/flagged_events?limit=50&refresh=false` | Fetch top N anomalies with SHAP explanations, LLM narratives, and ground truth |
| `GET` | `/health` | Backend liveness check (Redis status, LLM connectivity, model readiness) |
| `POST` | `/analyze_event` | Analyze a single security event (real-time mode) |
| `DELETE` | `/cache` | Flush the semantic cache (Redis + in-memory) |

## Getting Started

### Prerequisites

- **Node.js** ≥ 18
- **Backend running** on port 8000 (see `../backend/README.md`)

### Install & Run

```bash
# Install dependencies
npm install

# Start development server
npm run dev
```

The dashboard will be available at [http://localhost:3000](http://localhost:3000).

### Available Scripts

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `npm run dev` | Start Next.js dev server (Turbopack, memory-limited to 1 GB) |
| `build` | `npm run build` | Production build |
| `start` | `npm run start` | Serve production build |
| `lint` | `npm run lint` | Run ESLint |
| `format` | `npm run format` | Format all `.ts`/`.tsx` files with Prettier |
| `typecheck` | `npm run typecheck` | TypeScript type checking without emit |

### Adding shadcn/ui Components

```bash
npx shadcn@latest add <component-name>
```

Components are placed in `components/ui/`.

## Design Decisions

| Decision | Rationale |
|----------|-----------|
| **Dark theme only** | SOC analysts work in low-light environments for extended periods; dark UI reduces eye strain |
| **Single-page layout** | Sidebar + main panel eliminates navigation friction during active incident investigation |
| **No automated enforcement** | System cannot act without human confirmation — eliminates catastrophic false positives (e.g., locking a CFO during earnings) |
| **Memory-limited dev server** | `NODE_OPTIONS=--max-old-space-size=1024` prevents Turbopack from consuming 10–15 GB RAM alongside the ML backend |
| **Severity color coding** | Red (Critical), Amber (High), Yellow (Elevated), Emerald (Normal) — follows SOC industry conventions |
